Intro To Data Structures

Data structures are specialized ways of organizing and storing data so that it can be accessed, modified, and managed efficiently. Instead of treating data as a simple collection of values, data structures provide a framework that defines how those values relate to each other and how operations—such as searching, inserting, or deleting—are performed. Choosing the right data structure can significantly impact the performance and scalability of a program.

In computer science, data structures serve as the foundation for building algorithms and solving complex problems. Different structures are optimized for different tasks: arrays allow fast indexing, linked lists enable flexible memory usage, and trees and graphs help represent hierarchical or networked relationships. By understanding data structures, developers can write more efficient code, reduce computational costs, and design systems that handle large amounts of data effectively.

Why Data Structures Matter

  • Improve efficiency of algorithms
  • Organize data for better readability and maintainability
  • Enable faster data access and modification
  • Support complex problem-solving (e.g., searching, sorting, networking)
  • Form the backbone of software systems and applications
Arrays

Arrays are one of the most fundamental data structures in computer science. They store elements in contiguous memory locations, allowing for efficient indexing using an integer position. Because of this layout, accessing an element by index is extremely fast—constant time, or O(1). Arrays are widely used when you need predictable performance and fixed-size collections.

However, arrays come with tradeoffs. Their size is typically fixed, meaning resizing requires allocating new memory and copying elements over. Insertions and deletions in the middle of an array can be costly, as elements must be shifted. Despite these limitations, arrays serve as the backbone for many higher-level data structures.

Key Characteristics

  • Key-value storage
  • Average O(1) operations
  • Uses hash functions
  • Collision handling required
  • Backbone of dictionaries/maps

Example: Basic array usage in Python (list behaves like dynamic array)


        arr = [10, 20, 30, 40, 50]

        # Access
        print(arr[2])  # 30

        # Update
        arr[1] = 25

        # Insert (O(n))
        arr.insert(2, 15)

        # Delete (O(n))
        arr.pop(3)

        print(arr)
Linked Lists

A linked list is a linear data structure where elements (nodes) are connected via pointers rather than stored contiguously. Each node contains a value and a reference to the next node in the sequence. This design allows for efficient insertions and deletions without shifting elements.

Unlike arrays, linked lists do not support fast random access. Traversing to a specific element requires iterating through the list, making access O(n). However, their flexibility in dynamic memory allocation makes them useful for applications where frequent modifications are required.

Key Characteristics

  • Dynamic size
  • Efficient insertion and deletion (O(1) with pointer)
  • No random access (O(n) traversal)
  • Uses extra memory for pointers
  • Can be singly or doubly linked

Example: Singly Linked List


          class Node:
          def __init__(self, data):
              self.data = data
              self.next = None

          # Create nodes
          head = Node(10)
          second = Node(20)
          third = Node(30)

          # Link nodes
          head.next = second
          second.next = third

          # Traverse
          current = head
          while current:
              print(current.data)
              current = current.next
Stacks and Queues

Stacks and queues are abstract data types that define specific ordering principles for element access. A stack follows the Last In, First Out (LIFO) principle, meaning the last element added is the first to be removed. Common applications include function call stacks and undo operations.

Queues follow the First In, First Out (FIFO) principle, where elements are processed in the order they are added. They are commonly used in scheduling systems, buffering, and breadth-first search algorithms. Both structures can be implemented using arrays or linked lists.

Key Characteristics

  • Stack: LIFO ordering
  • Queue: FIFO ordering
  • O(1) insertion and removal
  • Used in recursion, scheduling, and traversal
  • Can be implemented with multiple underlying structures

Stack example


          stack = []

          stack.append(10)
          stack.append(20)
          stack.append(30)

          print(stack.pop())  # 30

Queue example


          from collections import deque

          queue = deque()

          queue.append(10)
          queue.append(20)
          queue.append(30)

          print(queue.popleft())  # 10
Trees

Trees are hierarchical data structures composed of nodes connected by edges. The top node is called the root, and each node may have child nodes. Trees are commonly used to represent hierarchical relationships, such as file systems or organizational structures.

A special type of tree, the binary tree, limits each node to at most two children. Variants like binary search trees (BSTs) maintain ordering properties that enable efficient searching, insertion, and deletion operations—typically O(log n) in balanced cases.

Key Characteristics

  • Hierarchical structure
  • Root, parent, and child relationships
  • Traversal methods (inorder, preorder, postorder)
  • Balanced vs unbalanced trees
  • Used in databases, file systems, and search algorithms

Example: Binary Tree Node and traversal


          class TreeNode:
          def __init__(self, value):
              self.value = value
              self.left = None
              self.right = None

          # Create tree
          root = TreeNode(10)
          root.left = TreeNode(5)
          root.right = TreeNode(15)

          # Inorder traversal
          def inorder(node):
              if node:
                  inorder(node.left)
                  print(node.value)
                  inorder(node.right)

          inorder(root)
Graphs

Graphs consist of a set of nodes (vertices) connected by edges. They are used to model relationships between entities, such as social networks, maps, and recommendation systems. Graphs can be directed or undirected, weighted or unweighted.

Traversal algorithms like Depth-First Search (DFS) and Breadth-First Search (BFS) are essential for exploring graphs. Graphs can be represented using adjacency lists or adjacency matrices, each with different performance tradeoffs depending on the use case.

Key Characteristics

  • Nodes (vertices) and edges
  • Directed or undirected
  • Weighted or unweighted
  • Traversal algorithms (DFS, BFS)
  • Represented via adjacency list or matrix

Example: Graph using adjacency list


          graph = {
            'A': ['B', 'C'],
            'B': ['D'],
            'C': ['D'],
            'D': []
          }

          # DFS traversal
          def dfs(node, visited=set()):
              if node not in visited:
                  print(node)
                  visited.add(node)
                  for neighbor in graph[node]:
                      dfs(neighbor, visited)

          dfs('A')
Hash Tables

Hash tables store key-value pairs and use a hash function to compute an index where the value is stored. This allows for extremely fast average-case operations O(1) for insertion, deletion, and lookup.

Collisions occur when multiple keys map to the same index. These are handled using techniques like chaining (linked lists) or open addressing. Hash tables are widely used in dictionaries, caches, and database indexing.

Key Characteristics

  • Key-value storage
  • Average O(1) operations
  • Uses hash functions
  • Collision handling required
  • Backbone of dictionaries/maps

Example: Hash table using Python dictionary


          hash_table = {}

          # Insert
          hash_table["apple"] = 5
          hash_table["banana"] = 3

          # Access
          print(hash_table["apple"])  # 5

          # Delete
          del hash_table["banana"]

          print(hash_table)
